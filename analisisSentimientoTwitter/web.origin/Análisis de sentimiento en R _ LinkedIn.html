Análisis de sentimiento en R

En el contexto de la minería de textos, el análisis de sentimiento consiste en el estudio de la información contenida en un texto para identificar reacciones, actitudes, contexto y emociones, clasificándola de forma masiva y automática. Cabe resaltar que dicha clasificación se basa en relaciones estadísticas y de asociación, mas no en análisis lingüístico.

El análisis de sentimiento está basado en lexicons. Un lexicon es, en términos simples, un vocabulario (diccionario). En el presente contexto un lexicon es un conjunto de palabras puntuadas positiva y negativamente. En este artículo se utilizará el paquete syuzhet dado que incluye una combinación de los lexicons más comunes como nrc, bing y afinn.

Es posible realizar un análisis de sentimiento a todo tipo de texto, libros, correos electrónicos, discursos, publicaciones en redes sociales, entre otros. En esta ocasión realizaremos un análisis básico, para lo cual utilizaremos la API de Twitter, dado que es de libre acceso (sólo basta con tener una cuenta en la mencionada red social).

Analizaremos los "sentimientos" que expresan los usuarios (en Twitter) de una de las principales marcas de telefonía móvil en Perú. Pudiendo extenderse este análisis a otro tipo empresas, marcas, políticos, entre otros; siempre y cuando tengan bastante actividad en la red social.

¡Comencemos!

Consiguiendo acceso a la API de Twitter

Utilizaremos el paquete twitteR el cual requiere cuatro claves (Keys) para poder minar Twitter. Estas claves son: consumer_key, consumer_secret, access_token, access_secret.

Estas claves se consiguen al crear un App para Twitter. Los pasos para obtenerlos son bastante sencillos:

  1. Crea una cuenta de Twitter, mucho mejor si ya tienes una.
  2. Crea una nueva App en: https://apps.twitter.com/
  3. Necesitarás una dirección URL personal (página web, blog o similares), en caso no tengas una, puedes crear una rápidamente en Wordpress.
  4. Consigue tus claves en la pestaña: Keys and Access Tokens
  5. Finalmente ya podemos usarlas en el paquete de R.

Importando datos de la cuenta de Twitter a analizar

Se pueden importar datos de dos maneras distintas:

  1. Vía usuario de twitter, por ejemplo: @Falabella_pe
  2. Término de búsqueda, por ejemplo: "Saga Falabella"

En el primer caso el paquete importará sólo los tweets que publique el usuario @Falabella_pe, mientras que en el segundo caso se importarán todos los tweets que tengan el término de búsqueda definido, independientemente del usuario que publique el tweet.

Dado que queremos analizar las opiniones de los usuarios, utilizaremos el segundo caso. De todas maneras se pondrá el código para el primer caso para los fines que crean convenientes.

#Caso 1: Importando tweets según usuario
#La función userTimeline tiene dos argumentos principales:
#user y n
#user hace referencia al usuario
#y n al número máximo de tweets que se desee importar
tweets <- userTimeline(user="@ClaroPeru", n=200)

#Caso 2: Importando tweets según término de búsqueda
#La función searchTwitter tiene varios argumentos, entre los
#principales, destacan: searchString, n, lang, geocode
#searchString se refiera a la cadena de texto que se buscará en los tweets
#n, al número máximo de tweets que se desee importar
#lang, al idioma en el que se desean los tweets, y
#geocode, con tres sub argumentos indica la latitud, longitud y el radio
#para poder focalizar mejor la importación de tweets, la latitud y longitud
#en el código a continuación se ubica en la Plaza Dos de Mayo (Lima, Perú)
tweets <- searchTwitter("@ClaroPeru", n=200,
                        lang="es",
                        geocode = '-12.046374,-77.042793,1500km')

#Verificando cantidad de tweets importados
n.tweet <- length(tweets)
n.tweet

#Convirtiendo los tweets en un data frame
tweets.df <- twListToDF(tweets) 
head(tweets.df)

#Quitando los links en los tweets
tweets.df2 <- gsub("http.*","",tweets.df$text)
tweets.df2 <- gsub("https.*","",tweets.df2)

#Quitando los hashtags y usuarios en los tweets
tweets.df2 <- gsub("#\\w+","",tweets.df2)
tweets.df2 <- gsub("@\\w+","",tweets.df2)

#Quitando los signos de puntuación, números y textos con números
tweets.df2 <- gsub("[[:punct:]]","",tweets.df2)
tweets.df2 <- gsub("\\w*[0-9]+\\w*\\s*", "",tweets.df2)

Análisis de sentimiento

Para clasificar las palabras de cada tweet se utilizará la función get_nrc_sentiment que tiene como argumentos a: char_v que hace referencia al vector de caracteres que será analizado y a language que hace referencia al idioma en el que se encuentra el texto de char_v.

La función get_nrc_sentiment devuelve un data frame en el cual cada fila representa cada tweet y cada columna representa cada una de ocho "emociones": anger, anticipation, disgust, fear, joy, sadness, surprise, trust, en las cuales son clasificadas las palabras de cada tweet; así como el puntaje positivo o negativo.

#Transformamos la base de textos importados en un vector para
#poder utilizar la función get_nrc_sentiment
palabra.df <- as.vector(tweets.df2)

#Aplicamos la función indicando el vector y el idioma y creamos
#un nuevo data frame llamado emocion.df
emocion.df <- get_nrc_sentiment(char_v = palabra.df, language = "spanish")

#Unimos emocion.df con el vector tweets.df para ver como
#trabajó la función get_nrc_sentiment cada uno de los tweets
emocion.df2 <- cbind(tweets.df2, emocion.df)
head(emocion.df2)

#Creamos un data frame en el cual las filas serán las emociones
#y las columnas los puntajes totales

#Empezamos transponiendo emocion.df
emocion.df3 <- data.frame(t(emocion.df))

#Sumamos los puntajes de cada uno de los tweets para cada emocion
emocion.df3 <- data.frame(rowSums(emocion.df3))

#Nombramos la columna de puntajes como cuenta
names(emocion.df3)[1] <- "cuenta"

#Dado que las emociones son los nombres de las filas y no una variable
#transformamos el data frame para incluirlas dentro
emocion.df3 <- cbind("sentimiento" = rownames(emocion.df3), emocion.df3)

#Quitamos el nombre de las filas
rownames(emocion.df3) <- NULL

#Verificamos el data frame
print(emocion.df3)
